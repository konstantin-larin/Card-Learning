Card Learning

Цель приложения

Конспектирование и повторение учебных материалов по карточкам

Основные функции

  • Материалы хранятся по иерархии: Предмет - Тема - Карточка.
  • Пользователь может создать предмет, добавить в него темы, а в тему добавить карточки с информацией
  • Возможности карточки:
    • Добавление названия и описания
    • Добавлять в содержимое текст, а также менять его величину, жирность, стиль написания
    • Добавлять в содержимое изображения

Приложение будет кроссплатформенным, но исключительно на ПК

Интерфейс пользователя

  • Приложение разделено на главные страницы:
    • Страница выбора предмета
    • Страница выбора темы
    • Страница с карточками
  • Каждая страница разделена на шапку и основное содержимое.
  • В шапке слева находится ссылка на предыдущий экран, а по центру заголовок.
  • Заголовок зависит от текущей страницы:
    • Выбор предмета - заголовок является названием программы
    • Выбор темы - заголовок является выбранным предметом
    • Карточки - заголовок является выбранной темой
  • Страница выбора предмета. Это страница выбора, создания, удаления предметов.
  • Страница выбора темы. После выбора предмета пользователь переходит на страницу выбора темы. Темам в списке автоматически присваивается упорядоченный номер.
  • Темам и предметам можно выбирать цвет. Цвет предмета отображается в качестве фона приложения. Цвет карточек такой же, как у их темы.
  • Страница с карточками. Представляет собой ряды карточек, идущих по порядку слева направо и вниз.
  • Для редактирования или создания предмета или темы используется модальное окно с формой для заполнения данных.
  • Возможность добавления темы или предмета представлена в виде кнопки справа от списка тем или предметов соответственно. После клика всплывает модальное окно с формой заполнения данных.
  • Интерфейс карточки:
    • Сама карточка представляет полупрозрачный скругленный прямоугольник.
    • Название карточки находится под ней. Длина названия ограничена до 20 символов
    • При клике на карточку открывается ее модальное окно
    • Главным содержимым окна является текстовое поле с панелью инструментов вверху (стиль текста, вставка картинки)
    • Над текстовым полем есть текстовое поле описания
    • Над текстовым поле описания поле заголовка. Поле заголовка стилистически отличается - оно без фона.
    • Справа вверху экрана большие иконки "Удалить", "Закрыть".
  • В упорядочивании списка предметов, тем и карточек используется механизм drag'n'down (захват и перемещение мышью).

Стэк

  • Я буду писать этот проект на веб-технологиях.
  • В качестве backend буду использовать встроенную браузерную бд - indexedDB, но с использованием обертки над промисами
  • В качестве frontend буду использовать html, css, js с использованием web-компонентов
  • Сделаю приложение десктопным с помощью electron.
  • Текстовый редактор у карточки будет сделан с помощью tinyMCE

Хранение данных

Так как проект небольшой, все данные будут храниться локально: либо в папках приложения, либо в indexedDB
